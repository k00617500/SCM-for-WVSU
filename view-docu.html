Project Overview

LitLatté POS Suite is a smart, seamless, and AI-powered Point-of-Sale & Café Suite System designed specifically for BookLatté Iloilo.

It centralizes ordering, inventory tracking, sales analytics, and customer engagement. The system integrates demand forecasting, real-time stock monitoring, and operational insights to help baristas, cashiers, and managers streamline workflows and improve service quality.

With automated syncing and real-time analytics, BookLatté can reduce waste, anticipate peak hours, track best-selling items, and optimize café operations. LitLatté ensures accuracy, speed, and reliability across all café processes—from ordering to payment handling, inventory updates, staff activity logs, and menu performance tracking.

Clone the repository using the following commands:

System Features

  • Centralized Order Management: Unified interface for order taking, billing, and transaction processing.
  • Smart Inventory Management: Tracks ingredient usage in real-time and issues low-stock alerts automatically.
  • AI-Powered Demand Forecasting: Predicts peak hours and best-selling items to optimize staffing and stock levels.
  • Sales and Performance Monitoring: Ensures transparent, tamper-proof records for audits and accountability.
  • Product Menu & Pricing Management: Easily update items, categories, and prices across the POS system.

User Manual & Instructions

1. Account Setup

Admin Login: Administrators access the system using master credentials issued upon deployment.

Cashier / Employee Accounts: Accounts are created by the administrator. Employees receive credentials directly from the manager. Self-registration is not allowed.

New Employees: Admins register new employees and issue initial credentials.

Password Reset: Use "Forgot Password" and enter your registered email to receive a secure reset link.

2. POS Operations Workflow

  1. Navigate to POS > New Order.
  2. Select menu items, add add-ons or custom notes, and confirm quantities.
  3. Proceed to checkout and finalize the transaction.

3. Order Processing

Completed orders automatically deduct ingredients from inventory and queue in the Barista Station. Baristas update order status: Preparing, Completed, or Served. All actions are logged per employee.

4. Inventory Management

The system monitors stock in real-time, checks for low or out-of-stock items, and forecasts supply needs. Admins are notified of discrepancies.

5. AI-Powered Demand Forecasting

Analytics Dashboard provides insights on best-selling items, peak hours, inventory trends, and allows CSV/PDF exports.

6. Product Menu & Pricing Management

Manage products, categories, prices, discounts, and seasonal offerings. Changes sync across all POS terminals in real-time.

7. Security & Roles

Role Permissions
Admin / Manager Full access to system settings, user management, employee credentials, and POS configuration
Cashier / Staff Access assigned POS terminals, process sales, manage orders, view relevant reports

8. Troubleshooting

  • Database Connection Failed: Check firebaseConfig.js credentials.
  • Sales/Inventory not syncing: Verify internet and Firebase status.
  • UI issues: Clear browser cache or use Chrome/Edge.
  • Receipt missing: Confirm POS user has write permissions in Firestore.

9. Feedback & Contributions

Report bugs via GitHub Issues. Submit improvements via Pull Request. Contact developers listed above for setup assistance.

System Architecture

  • Frontend: HTML, CSS, JavaScript for responsive interfaces
  • Backend: PHP modular structure handling business logic and authentication
  • Database: Firebase Realtime Database / Firestore for real-time tracking
  • Security: Role-based access control, HTTPS for data transmission

Data Flow

  1. User interacts with frontend (HTML, CSS, JS)
  2. Transaction/order data sent to PHP backend
  3. Backend logs transaction and updates Firebase in real-time
  4. Receipt generated and inventory updated automatically
  5. Analytics Dashboard visualizes sales, inventory, and customer activity
